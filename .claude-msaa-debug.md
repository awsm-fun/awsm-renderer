# MSAA Stage 3 fix — session 4 (May 27, 2026)

## Outcome

The Stage 3 classify → per-shader edge_resolve → final_blend dispatch
chain is **active end-to-end** and producing correct MSAA-resolve
output. The legacy `msaa_resolve_samples` / `msaa_process_sample` /
`msaa_apply_instance_tint` cross-shader resolve helpers are
**deleted** from the codebase. This is the Stage-3.2 work the plan
doc claimed done but had never actually been performed.

Verified visually on `MorphStressTest` (8 morphing capsules against
photo-studio skybox) + `Fox` — capsule silhouettes show smooth
4-sample-averaged transitions, matching `main` quality.

## Why this matters

The legacy `msaa_process_sample` carried a runtime `switch` over every
registered `shader_id`. Each new dynamic material registered by an
embedding app would grow the switch by one more branch, and that
branch's shading body would be inlined into **every** primary opaque
pipeline's SPIR-V — including pipelines specialized for unrelated
shader_ids. That's an O(N) bloat in the number of dynamic shaders.

Stage 3's per-shader-id specialization fixes this: each
`edge_resolve` pipeline is templated for one shader_id and only
contains that shader_id's shading body. Adding a dynamic material
doesn't grow existing pipelines.

## Code shape now

**Primary opaque (`material_opaque/.../compute.wgsl`):**
- Shades sample-0 directly for the dispatched shader_id only.
- For all-skybox pixels: PBR-specialized pipeline writes the skybox
  base color.
- For sample-0-is-skybox + other-samples-geometry edge pixels: writes
  skybox base color (the Stage 3 dispatch overwrites with the proper
  4-sample blend).
- No `msaa_resolve_samples` call, no cross-shader switch.

**Per-shader edge_resolve (`material_opaque/.../edge_resolve.wgsl`):**
- Indirect-dispatched over classify-emitted entries.
- Each pipeline templated for one shader_id only (PBR / Unlit / Toon /
  Flipbook / dynamic).
- Reads `MsaaSampleTextures` via the retained
  `msaa_load_sample_textures` helper.
- Writes shaded sums into per-edge accumulator slots.

**Skybox edge_resolve + final_blend:**
- Unchanged from prior state. final_blend reads accumulator slots,
  averages, writes `opaque_tex`.

**`material_shading.wgsl` (helpers):**
- `MsaaResolveResult`, `MsaaSampleResult` structs: **deleted**.
- `msaa_apply_instance_tint`, `msaa_process_sample`,
  `msaa_resolve_samples`: **deleted** (~180 lines).
- `MsaaSampleTextures` struct + `msaa_load_sample_textures`:
  **retained** (used by per-shader edge_resolve).
- `compute_material_color` (PBR mip-mode dispatch helper):
  **retained**.

**`material_classify/.../compute.wgsl`:**
- Already shipped with the mesh-vs-mesh edge detector (`1a3f68b`)
  and the slot_map mirror increment (`compute.wgsl:237-238` pair).
- No changes this session.

**Render flow (`material_opaque/.../render_pass.rs`):**
- `render_edge_resolve` dispatch is **active** (no early-return).
- Per-shader edge_resolve fires for every registered material; the
  skybox dispatch handles uncovered samples; final_blend writes
  resolved colors to `opaque_tex`.
