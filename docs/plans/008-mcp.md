# 008 — MCP exposure + documentation: comprehensive pass

**Order:** eighth — after the feature/optimization work settles, so this pass documents
and exposes what actually exists (doing it earlier means redoing it).

## Why
Recurring failure mode (see MCP-BUGS.md history): the MCP either doesn't expose a
capability the editor has, exposes it with wrong/stale docs, or exposes it in a way
agents can't discover. Each instance costs a downstream session hours. This is a
systematic close-out, not another spot fix.

## Part 1 — Exposure audit (capability parity)
Build a parity matrix: every `EditorCommand` variant and every `EditorQuery` vs the MCP
tool surface (`packages/mcp/src/mcp.rs`). For each: exposed? partial-update where the
command supports it? readable back (get/snapshot), not just settable?
- Known gaps going in: no `get_post_process` (001), `ssr_temporal_weight` missing from
  `PostProcessParams` (001), view options readback (002), global shadows config (006
  Phase 0.5), instancing + prefab denotation (006 axes 4-5), texture export overrides
  (`SetTextureExport` — verify exposed + documented).
- Add an **automated parity test**: a native test that walks the `EditorCommand` /
  `EditorQuery` schema (they derive schemars) against a checked-in allowlist of
  "intentionally not exposed" variants, so a new command without an MCP tool (or an
  allowlist entry + rationale) fails CI. That makes this pass permanent instead of a
  one-off sweep.

## Part 2 — Documentation audit
- Every tool description accurate and complete: parameters, units, valid ranges,
  accepted string forms (the `"builtin"`/`"built_in_default"` asymmetry class of bug),
  partial-update semantics, and WHEN to use the tool (the 002 view-toggles guidance
  pattern: screenshot tool says "disable grid/gizmos first").
- Every `awsm://docs/*` resource re-read against current behavior: asset-workflows
  (env reset strings — stale now), screenshot workflow, import workflow, LOD/nanite
  bake docs. Fix all stale strings found in the 001 audit (`set_camera_clip` claiming
  0.1/10000 defaults, etc.).
- Error messages: every `invalid_params` names the valid values (the `get_kind_schema`
  unknown-selector treatment is the model — apply it everywhere).
- Output budgets: no tool may return unbounded output; verify pagination/variant
  selection exists everywhere it's needed (the `get_uv_layout`/`get_kind_schema`
  lessons), and that spill-to-file behavior is documented. Bulk bytes NEVER ride tool
  results — side-channel (`/png`/`/glb`/`/bundle`) + path + manifest, per the standing
  rule.

## Part 3 — Agent-workflow smoke test
Script a full agent session against a dev MCP (`task mcp-dev`, dev port): import →
material edit → environment → post-process (incl. SSR) → view toggles → screenshot →
save → export bundle → verify each step used only documented tools and no step needed
out-of-band knowledge. Every friction point found becomes a doc/tool fix in this plan.
Record the script so it can re-run after future changes.

## Acceptance
- Parity matrix committed (doc or test fixture); CI parity test green.
- Zero known stale tool descriptions/resources; MCP-BUGS.md items all reflected.
- The scripted agent session completes cleanly using only what the tools themselves
  document.
